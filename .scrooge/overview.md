Scrooge is a token-miserly coding agent that splits work between a costly SOTA planner (Scrooge) and a cheap tool-using executor (Cratchit) to minimize LLM spend while still solving complex coding tasks.

Scrooge never reads source files or writes code — he receives only deterministic briefs (a tree-sitter code map, a call graph, and a project overview) and produces terse numbered plans. Cratchit, equipped with file editing, shell, Python, Wolfram, and documentation tools, executes each plan step in a fresh session, so no step depends on another's working memory. Every execution is followed by a machine-generated verdict (git diffstat + format/test/lint checks); mechanical failures loop back to Cratchit without burning a Scrooge round. The `Orchestrator` rebuilds Scrooge's context each round from one-line digests of prior reports plus the latest plan and results, so the expensive model's prompt never grows with superseded history. A single-process `CodeMap` cache (invalidated by mtime+file-count) ensures the free deterministic infrastructure stays O(1) across repeated tool calls within a session.
