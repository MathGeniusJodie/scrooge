**Scrooge** is a token-miserly coding agent where an expensive SOTA model (Scrooge) plans from compact briefs while a cheap model (Cratchit) executes all file-reading, code-writing, and verification with a full tool belt.

The architecture is built around a single invariant: never burn LLM tokens on work a deterministic program can do. A tree-sitter-based code map (`codemap.rs`) extracts every symbol and the call graph at zero token cost, so Scrooge plans from compressed briefs without ever reading source files. Cratchit's briefing injects the code map, keyword-matched best practices (`practices.rs`), and a reusable-helper index (`helpers.rs`) deterministically — the model never fetches them. The orchestration loop (`agents.rs`) chains: deterministic context gathering → one Scrooge planning turn (with a KV-cache-friendly prefix) → Cratchit execution → mechanical check verification (`checks.rs`), with check failures retrying through Cratchit directly to avoid burning expensive Scrooge rounds. Scrooge is only told DONE when checks pass green. The same binary serves dual roles: a standalone CLI and an MCP stdio server (`mcp.rs`) where a Claude Code conversation plays Scrooge and delegates legwork to Cratchit. Every LLM call is bookkept in `accounting.rs` with per-agent token counts and a "shillings saved" metric computed against configurable Scrooge-model rates, making the economic thesis of the two-tier split visible in `.scrooge/ledger.json`.
