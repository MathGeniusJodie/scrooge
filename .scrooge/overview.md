**Scrooge** is a token-miserly two-tier coding agent where an expensive SOTA model (Scrooge) plans from compact briefs while a cheap model (Cratchit) executes all file-reading, code-writing, and verification with a full tool belt.

The architecture is built around a single invariant: never burn LLM tokens on work a deterministic program can do. A tree-sitter-based code map (`codemap.rs`) extracts every symbol and the call graph at zero token cost; a keyword-matched best-practices document (`practices.rs`) injects only relevant guidance; and a reusable-helper index (`helpers.rs`) prevents reinventing utilities. The orchestration loop (`agents.rs`) chains: deterministic context gathering → one Scrooge planning turn (with a KV-cache-friendly prefix rebuilt once per task) → Cratchit execution → mechanical check verification (`checks.rs`), with check failures retrying straight through Cratchit in the same live conversation to avoid burning expensive Scrooge rounds. Scrooge is only told DONE when checks pass green. The same binary serves dual roles: a standalone CLI with subcommands (`run`, `cratchit`, `ask`, `map`, `check`, `helpers`, …) and an MCP stdio server (`mcp.rs`) where a Claude Code conversation plays Scrooge and delegates legwork to Cratchit. A project overview (`.scrooge/overview.md`), written by Cratchit on first use and refreshed after structural changes, is injected verbatim into every briefing so both agents share the same architectural understanding. Every LLM call is bookkept in `accounting.rs` with per-agent token counts and a "shillings saved" metric computed against configurable Scrooge-model rates (`.scrooge/rates.toml`), making the economic thesis of the two-tier split visible in `.scrooge/ledger.json`.
